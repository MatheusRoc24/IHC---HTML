O que são Media Queries?

Como realmente ter certeza de que nossa página web está funcionando corretamente em todos os dispositivos? Esta com certeza é a pergunta do milhão, pois se não utilizarmos os recursos corretamente será muito improvável que vamos conseguir responder esta pergunta.

Atualmente sabemos que o uso de dispositivos móveis, como o smartphone, é o que domina o mundo quando falamos em acesso à informação, e por isso o requerimento de criar páginas acessíveis para todos os dispositivos não é mais uma opção. É sim uma obrigação.

Neste artigo vamos entender o básico necessário para utilizarmos Media Query, um dos recursos mais poderosos quando falamos em criar layouts responsivos, ou seja, layouts que funcionam em todos os dispositivos.

Entendendo o design responsivo

Para tentar resumir um pouco, design web responsivo é o forma de uso do HTML e CSS para a criação de uma aplicação (ou página) web que irá funcionar corretamente em todos os dispositivos, independentemente do tamanho da tela de cada dispositivo.

Todos os elementos que temos em tela, como botões, imagens, funcionalidade e demais recursos devem ser vistos e ter a mesma funcionalidade em monitores gigantescos e também em telas menores como a de um smartphone.

E é esse o pensamento de responsividade que todos nós, desenvolvedores de softwares, devemos ter ao criar qualquer tipo de página ou aplicação web. É válido salientar também, que existem diversas técnicas além do Media Query que podemos utilizar para nos auxiliar no desenvolvimento de páginas responsivas.

  • Usar unidades relativas ao invés de absolutas (por exemplo, usar % ao invés de px);
  • Configurar corretamente o viewport/scale da página;
  • Utilizar recursos de acessibilidade de páginas.

Porém, neste artigo vamos focar em Media Query pois na minha opinião é a melhor maneira de começarmos.

O que são Media Queries

Media Query é um recurso do CSS que veio juntamente com a versão CSS3. Com este recurso, desenvolvedores tem a possibilidade de modificar o comportamento da página de acordo com o tamanho (width e height) das telas, ou seja, mediante a diferentes tamanhos de telas, a página terá diferentes comportamentos.

Utilizando Media Query você terá acesso a uma espécie de Framework para escrever o código somente uma vez e utiliza-lo em toda a sua aplicação/página. Logicamente, se estamos falando somente de uma página em específico, talvez você não veja tanta diferença assim, mas no momento que você trabalhar em uma aplicação que controla dezenas de páginas, você verá uma mudança gigantesca no uso do Media Query.

Essa fato se dá pois ao usar Media Query você estará levando em consideração alguns fatores, como mencionei width e height, mas muito além disso, você deverá levar em consideração a estrutura, localização, links, posições, dentre outros.

A estrutura do Media Query

@media media-type and (expression) {
  /*CSS code*/
}

A estrutura padrão que devemos utilizar é a estrutura citada acima, pois ela contém alguns recursos que irão facilitar a mudança de comportamento dos nossos elementos. Vamos analisar melhor:

  • @media é a palavra-chave. Desta maneira que nossa aplicação consegue identificar que a partir desta palavra terá uma mudança de comportamento. Não é possível utilizar sem esta palavra-chave.
  • media-type é a forma de definir qual é o tipo de tela que desejamos criar o comportamento. Pode ser screen, handheld, print, all…é a forma que temos para direcionar o comportamento para uma tela em específico.
  • expression é realmente a condição que devemos colocar para aplicar o nosso novo comportamento, por exemplo, max-width: 400px. Sempre que o tamanho da nossa tela estiver dentro da condição citada, o novo comportamento irá ser aplicado.
@media screen and (max-width: 450px) {
  body {
    background-color: #004aad;
  }
}

No exemplo acima, estamos alterando o comportamento do body, aplicando uma cor de fundo diferente. Sempre que o máximo de largura da tela for 450px, a nova cor de fundo será aplicada.

Por padrão, podemos utilizar também sem especificar o media-type, aplicando o comportamento para todas as telas, mediante a expressão que definimos.

@media and (max-width: 450px) {
  body {
    background-color: #004aad;
  }
}

Em que lugar escrever as Media Queries?

Sabemos que o uso da Media Query é realmente para alterar um comportamento de CSS. Com base nisso, devemos sempre escrever a Media Query dentro no nosso arquivo CSS, ou no lugar onde definimos os estilos CSS.

Basicamente, existem três formas de definição do CSS:

  • CSS Externo: este com certeza é o melhor uso que podemos ter. Devemos definir um arquivo CSS externo ao nosso HTML e somente chamar este CSS dentro do HTML. É uma forma muito mais organizar e limpa de trabalhar com arquivo de estilos;
  • CSS Interno: podemos também utilizar o CSS dentro do HTML através da tag <head>, onde podemos utilizar a tag <style> e colocar todas as classes ou id’s referente ao CSS;
  • CSS inline: também podemos utilizar diretamente na <div> ou qualquer outra tag. Para fazer isso, basta digita style=”alterações” e colocar todas as alterações necessárias do CSS;

Não importa qual a forma que estamos utilizando para vincular o CSS no nosso HTML. De qualquer maneira podemos utilizar Media Queries dentro do CSS para alterar qualquer comportamento desejado.

Media Query com Links

Porém, também existe uma alternativa caso você não queira vincular a sua Media Query dentro do CSS. Esta alternativa inclui utilizar Media Queries diretamente na tag <link> do seu arquivo HTML. Desta maneira, você não precisa criar um arquivo CSS somente para a sua Media Query.

<link
    rel="stylesheet"
    media="(max-width:800px) and (min-width:451px)"
    href="tablet.css"/>
<link rel="stylesheet" media="(max-width:450px)" href="smartphone.css" />

Desta maneira toda a lógica que você precisa para usar o Media Query ficará dentro da tag <link>, o que dependendo do cenário, poder ser até mas produtivo para você.

É válido salientar que eu não recomendo isso para projetos maiores, pois o uso desta técnica pode poluir e muito o seu arquivo HTML. Mas é importante conhecermos que existe essa possibilidade que pode ser mais produtiva se for um projeto mínimo.

Demonstração Prática de Media Queries

Redimensione a tela para ver a mágica!

Largura da tela: --px
Largura da caixa: --px

Breakpoints

  • ≤ 480px → mobile
  • 481–768px → tablet
  • 769–1024px → laptop
  • ≥ 1025px → desktop

@media (max-width: 480px){
    .responsive-demo{
        width: 100%;
        background-color: #081a84;
    }
}

@media (min-width: 481px) and (max-width: 768px){
    .responsive-demo{
        width: 80%;
        background-color: #1033ac;
    }
}

@media (min-width: 769px) and (max-width: 1024px){
    .responsive-demo{
        width: 60%;
        background-color: #194dd5;
    }
}

@media (min-width: 1025px){
    .responsive-demo{
        width: 40%;
        background-color: #2166fd;
    }
}
